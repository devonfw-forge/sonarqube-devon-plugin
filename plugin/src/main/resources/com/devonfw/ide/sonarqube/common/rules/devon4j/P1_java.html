This rule checks that the naming of packages adheres to the devonfw package conventions. For more information on the devonfw naming conventions for packages, visit our coding conventions

Noncompliant Code Examples

    package com.devonfw.ide.sonarqube.persistence.api;

    package com.devonfw.ide.sonarqube;

    package com.devonfw.ide.sonarqube.common;

Compliant Solution

    package com.devonfw.ide.sonarqube.bookingmanagement.dataaccess.api;
