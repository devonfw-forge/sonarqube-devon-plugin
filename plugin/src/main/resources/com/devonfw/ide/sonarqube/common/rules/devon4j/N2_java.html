This rule checks types extending from Dao (Data-Access-Object) also start or end with Dao. Further if they are non-abstract they should carry the suffix and DaoImpl and implement an interface with their name excluding the Impl suffix.

Noncompliant Code Examples

    public abstract class ExampleClass extends AbstractDao {}

    public class ExampleDaoImpl extends AbstractDao {}

Compliant Solutions

    public abstract class AbstractExampleDaoImpl extends AbstractDao {}

    public class CustomerDaoImpl extends AbstractDao implements CustomerDao {}
