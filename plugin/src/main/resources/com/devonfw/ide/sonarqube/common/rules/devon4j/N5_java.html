This rule checks that non-abstract types inheriting from Uc (Use-Case) start or end with Uc. Further if they are non-abstract they should start with Uc, carry the suffix Impl and implement an interface with their name excluding the Impl suffix.

Noncompliant Code Examples

    public class UcExampleImpl extends AbstractUc {}

    public abstract class ExampleClass extends AbstractUc {}

Compliant Solution

    public class UcExampleImpl extends AbstractUc implements UcExample {}
